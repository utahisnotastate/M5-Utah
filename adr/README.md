# ADRs (Architecture Decision Records)

This folder captures major technical and process decisions.

## Why ADRs

- Preserve decision context as the project evolves
- Clarify tradeoffs for new contributors
- Reduce repeated debates and inconsistent direction

## ADR format

Each ADR should include:

- Status (`proposed`, `accepted`, `deprecated`, `superseded`)
- Context
- Decision
- Consequences
