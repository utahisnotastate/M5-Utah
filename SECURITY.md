# Security Policy

## Supported versions

This project is currently pre-1.0. Security fixes are applied on the default branch.

## Reporting a vulnerability

Please do not open a public issue for suspected security vulnerabilities.

Instead:

1. Share details privately with maintainers.
2. Include reproduction steps, impact, and affected files.
3. Allow time for triage and coordinated disclosure.

## Scope

- Host runtime input parsing
- Serial protocol safety and fault handling
- Dependency vulnerabilities
- Supply-chain risks in CI and releases
