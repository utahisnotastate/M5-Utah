# Non-Technical User Guide

## What you get

- One stable way to run M5Stack workflows
- Less confusion about which repository/library to use
- Faster support and onboarding

## Typical daily flow

1. Connect device
2. Start approved script
3. Observe output on device
4. Stop script when done

## Why this is simpler

- fewer moving parts
- one standard workflow
- faster iteration without constant firmware reflashing
