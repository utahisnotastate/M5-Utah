# M5Stack 员工迁移手册

## 摘要

Utah Flux Studio 是**默认面向客户的产品**。旧的分模块仓库在新开发中应转为参考/桥接/退役。

## 员工教程

[员工教程](tutorials/employee-tutorial.md)

## 迁移理由

| 指标 | 旧方式 | Utah Flux |
|------|--------|-----------|
| 首次演示 | 数小时 | 数分钟 |
| 课堂支持 | 编译错误多 | 可视化项目 |
| 透明度 | C++ 难读 | `.flux.json` + 注册表 |

## 30-60-90 天

- **0–30：** 冻结旧仓库新 API、发布中英入门、统一演示固件
- **31–60：** 迁移高频流程为模板、补齐注册表、支持培训
- **61–90：** 强制新功能只进 Utah Flux、发布 KPI、旧库退场时间表

## 仓库状态

核心（本仓库）/ 桥接 / 参考 / 退役

## KPI

- 新功能经 Utah Flux 交付占比
- “该用哪个库”类支持单
- 课堂首次 Play 时间
