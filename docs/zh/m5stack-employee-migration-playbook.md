# M5Stack 员工迁移手册

## 目标

将分散的按模块仓库模式在“新功能开发”中默认替换为统一运行时方案。

## 策略

- 新功能统一进入本仓库。
- 旧仓库进入退役/参考/桥接状态。
- 迁移期旧仓库仅做关键修复。

## 30-60-90 计划

- 0-30 天：冻结旧仓库新增扩展、完成 Unit 盘点、明确负责人
- 31-60 天：迁移高频工作流、补桥接层、完成回归验证
- 61-90 天：执行强制策略、发布 KPI、启动旧仓库退场

## 岗位要求

- 固件：保持运行时轻量稳定
- SDK/主机：构建类型化 API 与可复用 wire 流
- QA：验证契约兼容性与可靠性
- PM：跟踪迁移 KPI 与治理执行
