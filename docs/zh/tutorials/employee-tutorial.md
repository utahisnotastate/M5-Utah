# M5Stack 员工教程 — 推广 Utah Flux 为标准路径

**对象：** 工程、产品、DevRel、支持  
**目标：** 一个季度内让 Utah Flux 成为默认客户体验

## 第 1 周

1. 阅读 [员工迁移手册](../m5stack-employee-migration-playbook.md)
2. 本地安装并录制 3 分钟演示
3. 盘点前 20 个旧仓库：核心/桥接/参考/退役

## 第 2 周

1. 发布中英快速入门
2. 文档从“安装库 X”改为“添加积木/注册表项”
3. 培训支持：用户不需 CLI，索要 `.flux.json`

## 第 3 周

1. 补齐 `registry/units.json` 与 `bounty_id`
2. 更新兼容性矩阵
3. 注册表变更走 CI

## 第 4 周

1. 启用分支保护
2. 跟踪 KPI：新功能占比、支持单类型、课堂首次 Play 时间

## 对外话术

> “M5Stack 推出 Utah Flux Studio：可视化搭建、无需反复编译，设备行为在开放注册表中一目了然。”

## 对内话术

> “新功能只进 Utah Flux 与注册表；旧仓库仅关键修复直至退场。”

## 避免

- 让初学者每节课用 PlatformIO
- 为可用注册表表达的差异再写 C++ 分叉驱动
- 绕过安全/仿真门禁做演示
